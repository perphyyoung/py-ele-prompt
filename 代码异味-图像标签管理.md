# 图像标签管理代码异味

基于代码分析，我发现了以下 代码异味 ：

## 1. 重复的事件监听器绑定（高风险）
renderImageTags() 每次渲染都重新绑定删除事件，但没有先解绑旧的事件监听器：

```
// 每次渲染都重新绑定，可能导致内存泄漏或重复绑定
container.querySelectorAll('.tag-remove-btn').forEach(btn => {
  btn.addEventListener('click', async (e) => { ... });
});
```
改进方案 ：使用事件委托，将事件绑定到容器上。

## 2. 重复的标签过滤逻辑
多处重复过滤违单标签：

- renderImageTags() 中过滤
- 可能其他地方也需要过滤
改进方案 ：在 TagManager 中添加 getNormalTags() 方法统一过滤

## 3. 自动补全逻辑重复
图像和提示词的自动补全逻辑几乎相同，但分别实现：

- setupImageTagAutocomplete()
- setupPromptTagAutocomplete() （如果有的话）
改进方案 ：提取通用自动补全组件。

## 4. 魔法字符串
PromptManager.VIOLATING_TAG 在 TagManager 中使用，造成耦合：

```
const filteredTags = tags.filter(tag => tag !== PromptManager.VIOLATING_TAG);
```
改进方案 ：将违单标签常量移到 TagManager 或独立常量文件。

## 5. 回调地狱风险
addTagWithManager 中嵌套了多个条件判断和 toast 显示逻辑，可读性较差。

## 6. 缺少输入验证
setupImageTagAutocomplete() 中没有对输入进行 XSS 防护（虽然使用了 escapeHtml ）。
