# 数据模型

> 本文档描述 Prompt Manager 的数据库表结构和数据关系。

---

## 1. 核心表

### 1.1 提示词表 (prompts)

| 字段 | 类型 | 说明 |
| ------ | ------ | ------ |
| id | TEXT | 主键 |
| title | TEXT | 标题 |
| content | TEXT | 内容 |
| content_translate | TEXT | 翻译 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |
| is_deleted | INTEGER | 是否删除 |
| deleted_at | DATETIME | 删除时间 |
| is_favorite | INTEGER | 是否收藏 |
| is_safe | INTEGER | 安全评级 |
| note | TEXT | 备注 |

### 1.2 图像表 (images)

| 字段 | 类型 | 说明 |
| ------ | ------ | ------ |
| id | TEXT | 主键 |
| file_name | TEXT | 文件名 |
| stored_name | TEXT | 存储名 |
| relative_path | TEXT | 相对路径 |
| thumbnail_path | TEXT | 缩略图路径 |
| width | INTEGER | 宽度 |
| height | INTEGER | 高度 |
| file_size | INTEGER | 文件大小 |
| created_at | DATETIME | 创建时间 |
| is_deleted | INTEGER | 是否删除 |
| deleted_at | DATETIME | 删除时间 |
| is_favorite | INTEGER | 是否收藏 |
| is_safe | INTEGER | 安全评级 |
| note | TEXT | 备注 |
| gen_params | TEXT | 生成参数 (JSON) |

---

## 2. 关联表

### 2.1 提示词 - 图像关联

**prompt_image_relations** 提示词 - 图像关联表

| 字段 | 类型 | 说明 |
| ------ | ------ | ------ |
| prompt_id | TEXT | 提示词 ID |
| image_id | TEXT | 图像 ID |
| created_at | DATETIME | 创建时间 |

### 2.2 标签表

**prompt_tags** - 提示词标签
**image_tags** - 图像标签

| 字段 | 类型 | 说明 |
| ------ | ------ | ------ |
| id | TEXT | 主键 |
| name | TEXT | 标签名称 |
| type | TEXT | 标签类型（prompt/image） |
| created_at | DATETIME | 创建时间 |

### 2.3 标签关联表

**prompt_tag_relations** - 提示词 - 标签关联
**image_tag_relations** - 图像 - 标签关联

| 字段 | 类型 | 说明 |
| ------ | ------ | ------ |
| tag_id | TEXT | 标签 ID |
| target_id | TEXT | 目标 ID（prompt_id 或 image_id） |
| created_at | DATETIME | 创建时间 |

---

## 3. 数据关系

``` bash
prompts (提示词)
├── prompt_image_relations (关联图像)
│   └── images (图像)
└── prompt_tag_relations (关联标签)
    └── prompt_tags (标签)

images (图像)
├── prompt_image_relations (关联提示词)
│   └── prompts (提示词)
└── image_tag_relations (关联标签)
    └── image_tags (标签)
```

## 4. 索引建议

### 4.1 提示词表

```sql
CREATE INDEX idx_prompts_updated_at ON prompts(updated_at);
CREATE INDEX idx_prompts_created_at ON prompts(created_at);
CREATE INDEX idx_prompts_title ON prompts(title);
CREATE INDEX idx_prompts_is_deleted ON prompts(is_deleted);
CREATE INDEX idx_prompts_is_favorite ON prompts(is_favorite);
CREATE INDEX idx_prompts_is_safe ON prompts(is_safe);
```

```sql
CREATE INDEX idx_images_created_at ON images(created_at);
CREATE INDEX idx_images_is_deleted ON images(is_deleted);
CREATE INDEX idx_images_is_favorite ON images(is_favorite);
CREATE INDEX idx_images_is_safe ON images(is_safe);
```

### 4.3 关联表

```sql
CREATE INDEX idx_prompt_image_relations_prompt_id ON prompt_image_relations(prompt_id);
CREATE INDEX idx_prompt_image_relations_image_id ON prompt_image_relations(image_id);
CREATE INDEX idx_prompt_tag_relations_target_id ON prompt_tag_relations(target_id);
CREATE INDEX idx_image_tag_relations_target_id ON image_tag_relations(target_id);
```

## 5. 软删除机制

所有表都使用软删除机制：

- **is_deleted = 0** - 正常数据
- **is_deleted = 1** - 已删除数据（回收站）
- **deleted_at** - 删除时间戳

### 查询示例

```sql
-- 查询正常数据
SELECT * FROM prompts WHERE is_deleted = 0;

-- 查询回收站数据
SELECT * FROM prompts WHERE is_deleted = 1;

-- 彻底删除
DELETE FROM prompts WHERE is_deleted = 1;
```

## 6. 时间戳规范

### 存储格式

- 统一使用本地时间

### 前端显示

- 使用 `new Date(timestamp)` 转换为本地时间
- 格式化显示（YYYY-MM-DD HH:mm:ss）

## 7. 命名规范

### 数据库字段

- **snake_case**: `file_name`, `created_at`, `is_deleted`

### 前端数据

- **camelCase**: `fileName`, `createdAt`, `isDeleted`

### 转换函数

```javascript
// 数据库 → 前端
function toCamelCase(dbData) {
  return {
    fileName: dbData.file_name,
    createdAt: dbData.created_at,
    // ...
  };
}

// 前端 → 数据库
function toSnakeCase(frontendData) {
  return {
    file_name: frontendData.fileName,
    created_at: frontendData.createdAt,
    // ...
  };
}
```
