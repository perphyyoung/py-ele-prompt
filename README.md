# Prompt Manager

本地提示词管理工具，基于 Electron 开发。

## 项目概述

Prompt Manager 是一款本地提示词管理工具，基于 Electron 开发，用于帮助用户高效管理 AI 绘画/生成的提示词（Prompt）和关联图像。

### 目标用户

- AI 绘画爱好者
- 提示词收藏者
- 需要管理大量提示词和参考图像的用户

### 技术栈

- **前端框架**: Electron 28.0.0
- **后端**: Node.js 18.18+ + SQLite3
- **图像处理**: Sharp 0.33.0
- **UI**: 原生 HTML/CSS/JavaScript (ES2023)

### 代码架构

- **模块化**: ES Module 架构 (`import`/`export`)
- **常量管理**: 独立 Constants 类统一管理字符串字面量
- **保存机制**: 字段级变化追踪 + 防抖自动保存
- **导航统一**: ListNavigator 类提供一致的列表导航体验
- **上传架构**: 策略模式（DelaySaveStrategy/DirectSaveStrategy）+ 职责分离（ImagePreviewManager/UploadStrategies/ImageUploadService）

### 通用功能（适用于提示词和图像）

详见 [参考文档/通用功能.md](参考文档/通用功能.md)，包括：

- **标签管理系统** - 标签添加/筛选/组管理/拖拽/排序，单选组限制（Ctrl+点击多选时仍限制单选），特殊标签，标签筛选收起等
- **列表视图系统** - 视图切换（卡片/列表/紧凑），排序，列表多选，批量操作，搜索等
- **收藏系统** - 收藏/取消收藏，收藏筛选
- **回收站系统** - 软删除，恢复，彻底删除，卡片布局
- **数据管理系统** - 导入导出，数据目录，孤儿文件清理
- **系统功能** - 主题切换，Safe/NSFW 模式，侧边栏收起，系统托盘，页面记忆等
- **自动保存系统** - 防抖自动保存（800ms），字段级变化追踪，textarea 自动调整高度；关闭/导航时自动保存未保存的变更
- **对话框快捷键** - 确认对话框支持 Enter 确认、Esc 取消
- **安全评级系统** - 安全评级设置，双向联动（事件驱动架构，SafeRatingService 集中管理）
- **标签拖拽添加** - 支持从标签筛选区拖拽普通标签到卡片快捷添加标签
- **图像上传** - Safe File Dialogs 安全文件选择，多图上传，延迟保存策略（选择→预览→确认），事件委托优化
- **图像预填充** - 从图像详情界面添加提示词时，自动将当前图像预填充到参考图像区域

### 提示词特定功能

详见 [参考文档/特定功能.md#1-提示词特定功能](参考文档/特定功能.md#1-提示词特定功能)：

- **图像关联** - 添加本地图像，支持多图关联、首图显示、Hover 预览
- **特殊标签** - 多图、无图标识
- **安全评级联动** - 与关联图像双向联动（事件驱动架构）
- **图像标签显示** - 编辑界面图像缩略图显示标签

### 图像特定功能

详见 [参考文档/特定功能.md#2-图像特定功能](参考文档/特定功能.md#2-图像特定功能)：

- **提示词关联** - 多引用管理、解除关联、未引用识别、提示词预览
- **特殊标签** - 未引、多引标识
- **安全评级联动** - 与关联提示词双向联动（事件驱动架构）
- **图像导航** - 左右切换、键盘导航、全屏查看，导航按钮统一为圆形悬浮样式
- **文件名/备注编辑** - 图像详情界面支持编辑，防抖自动保存

### 其他

- **主题切换** - 明亮/暗黑模式（移至设置界面）
- **视图模式** - 支持 safe/nsfw 模式切换，控制内容显示范围
- **侧边栏收起** - 可收起侧边栏，扩大内容区域
- **系统托盘** - 最小化到系统托盘，支持托盘菜单重启应用
- **页面记忆** - 重启后恢复上次打开的页面
- **模态框尺寸** - 统一模态框尺寸类（large/medium/small），优化界面一致性
- **统一导航** - 全屏查看器、图像详情、提示词编辑三个界面使用统一的 ListNavigator 类和导航按钮样式
- **禁止二级跳转** - 从提示词详情界面进入图像详情页后，禁止再通过详情页按钮进入新的提示词详情界面，反之亦然；禁用按钮置灰并显示"已从详情界面跳转，禁止再次跳转"提示
- **详情界面切换优化** - 图像详情与提示词详情间单向切换采用 hide/show 机制，避免关闭重开，提升用户体验

## 启动

### 生产环境启动

使用 `start-hidden.vbs` 启动，不显示命令行窗口：

```bash
start-hidden.vbs
```

### 调试模式启动

使用 `start-debug.bat` 启动，会显示控制台日志：

```bash
start-debug.bat
```

### 开发环境启动（需安装 Node.js）

在项目根目录执行：

```bash
# 安装依赖
cnpm install

# 启动应用
cnpm start
```

## 设置

设置界面采用卡片式布局，包含以下功能：

- **外观** - 切换明亮/暗黑主题
- **数据存储** - 更改数据存储目录
- **数据管理**
  - **导入** - 从 JSON 文件导入所有提示词数据
  - **导出** - 将所有提示词数据导出为 JSON 文件
  - **导出孤儿文件** - 导出不在数据库中的图像文件到指定目录并清理
- **危险操作**
  - **清空所有数据** - 一键清空所有提示词、图像和标签数据（不可恢复）, 保留图像文件

## 数据存储

默认存储在应用目录下的 `py-data/` 文件夹中：

- **默认路径**：`{应用目录}/py-data/`
- 可通过设置界面更改数据存储路径: 设置-数据存储-数据目录-更改

### config.json 配置

项目默认包含 `config.json`，配置如下：

```json
{
  "dataDir": "./py-data"
}
```

如需更改数据存储路径，修改此文件即可; 推荐通过设置界面更改。

### 数据目录结构

``` bash
{数据目录}/
├── prompt-manager.db      # SQLite 数据库
├── images/{date}/                # 图像文件存储
│   ├── {stored_name}.jpg  # 原始图像
│   └── ...
└── thumbnails/{date}/          # 缩略图缓存
    ├── {stored_name}.jpg  # 缩略图
    └── ...
```

### 数据库结构

使用 SQLite 数据库存储所有数据，主要表结构：

| 表名 | 说明 |
| ------ | ------ |
| `prompts` | 提示词数据（标题、内容、创建时间、更新时间、收藏状态等） |
| `images` | 图像数据（文件名、路径、MD5、尺寸、创建时间、更新时间、收藏状态等） |
| `prompt_tags` | 提示词标签 |
| `image_tags` | 图像标签 |
| `prompt_tag_relations` | 提示词与标签的关联关系 |
| `image_tag_relations` | 图像与标签的关联关系 |
| `prompt_image_relations` | 提示词与图像的关联关系 |
| `db_version` | 数据库版本记录 |

### 数据库迁移

使用版本号机制管理数据库结构变更：

- 自动创建 `db_version` 表记录已应用的迁移版本
- 启动时检查当前版本，只执行未应用的迁移脚本
- 支持增量更新，避免重复执行已完成的迁移

### 图像数据结构

```json
{
  "fileName": "原始文件名.jpg",
  "storedName": "时间戳_随机数.jpg",
  "path": "绝对路径",
  "relativePath": "images/存储名.jpg",
  "thumbnailPath": "thumbnails/thumb_存储名.jpg",
  "md5": "原图像MD5哈希",
  "thumbnailMD5": "缩略图MD5哈希",
  "width": 1920,
  "height": 1080,
  "createdAt": "创建时间",
  "updatedAt": "更新时间",
  "isFavorite": false
}
```

**MD5 去重**：上传图像时会计算 MD5，如果已存在相同 MD5 的图像，则复用已有文件，避免重复存储。
