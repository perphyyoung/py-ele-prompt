# Project Rule - AI 开发约束

> **关联文档**: [需求文档](../../requirements.md) - 开发前请确认需求，开发后请验证是否符合需求

## 核心原则

1. **需求一致性** - 修改前确认不违反 requirements.md，修改后验证符合需求
2. **用户确认** - 添加功能时有更好的想法，先与用户商量
3. **操作提示** - 启动应用、安装依赖时提示用户操作
4. **修改计划** - 改代码前，先简述计划与影响范围; 如果有类似的功能已实现, 给出复用代码的利弊分析

## 技术约束

- 包管理: 使用 `cnpm` 替代 `npm`
- 命令行: 使用 `cmd` 而非 PowerShell
- 时区处理: 存储用 `new Date().toISOString()`，前端用 `new Date()` 转换
- 命名规范: 数据库 `snake_case`，前端 `camelCase`

## 界面规范

- UI 组件样式遵循 [design-system.md](../../design-system.md)

## 代码规范

- 控制台只输出英文
- 注释简洁
- Markdown 符合 markdownlint
- 更改与 README 不一致时提示更新

## 特殊指令

输入 **"总结修改"** 或 **"修改总结"** 时: 总结最近10条对话核心需求，用最简洁的文字, 输出可复制的 markdown 格式文本, 每行的格式为: `- 需求描述`
