# Project Rule - AI 开发约束

> **关联文档**: [需求文档](../../需求文档.md)

## 核心原则

1. **需求一致性** - 修改前确认不违反 需求文档.md，修改后验证符合需求
2. **用户确认** - 添加功能时有更好的想法，先与用户商量
3. **操作提示** - 启动应用、安装依赖时提示用户操作
4. **修改计划** - 改代码前，先简述计划与影响范围, 不需要出现代码块, 除非没有代码块说不清楚; 如果有类似的功能已实现, 给出复用代码的利弊分析

## 技术约束

- 包管理: 使用 `cnpm` 替代 `npm`
- 命令行: 使用 `cmd` 而非 PowerShell
- 时区处理: 存储用 `new Date().toISOString()`，前端用 `new Date()` 转换

## 界面规范

- UI 组件样式遵循 [设计规范.md](../../设计规范.md)

## 其他

- 修复完提醒我启动, 而不是直接启动 `npm start`
- 通过 logger.js 调试日志
