# Prompt Manager 需求文档

## 1. 项目概述

### 1.1 项目背景

Prompt Manager 是一款本地提示词管理工具，基于 Electron 开发，用于帮助用户高效管理 AI 绘画/生成的提示词（Prompt）和关联图像。

### 1.2 目标用户

- AI 绘画爱好者
- 提示词收藏者
- 需要管理大量提示词和参考图像的用户

### 1.3 技术栈

- **前端框架**: Electron 28.0.0
- **后端**: Node.js 18.18+ + SQLite3
- **图像处理**: Sharp 0.33.0
- **UI**: 原生 HTML/CSS/JavaScript (ES2023)

### 1.4 代码架构

- **模块化**: ES Module 架构 (`import`/`export`)
- **常量管理**: 独立 Constants 类统一管理字符串字面量
- **保存机制**: 字段级变化追踪 + 防抖自动保存
- **导航统一**: ListNavigator 类提供一致的列表导航体验

| 搜索提示词 | 数据库层面搜索，支持内容和备注搜索 | P0 |

## 3. 界面需求

> 所有界面元素遵循 [design-system.md](design-system.md) 的设计规范。

### 3.1 布局结构

```text
+------------------------------------------+
|  标题栏 (Prompt Manager)                 |
+----------+-------------------------------+
|          |                               |
|  侧边栏  |        内容区域               |
|  (导航)  |                               |
|          |  - 提示词管理                  |
|          |  - 图像管理                    |
|          |  - 设置                        |
|          |                               |
+----------+-------------------------------+
```

### 3.2 主要界面

#### 3.2.1 提示词管理界面

- **工具栏**: 新建、搜索、排序、视图切换、标签筛选
- **内容区**: 卡片网格或列表
- **卡片元素**: 首图背景、标题、内容预览、标签、收藏按钮、操作按钮

#### 3.2.2 提示词编辑界面

- **左侧**: 图像上传区域、图像预览列表
- **右侧表单**:
  - 提示词标题（*必需）
  - 提示词内容（*必需）
  - 提示词翻译
  - 提示词标签
  - 备注
  - 安全评级开关（与关联图像双向联动）

#### 3.2.3 图像管理界面

- **工具栏**: 上传、搜索、排序、视图切换、标签筛选
- **内容区**: 卡片网格或列表
- **卡片元素**: 图像缩略图、文件名、标签、收藏按钮、操作按钮

#### 3.2.4 图像上传界面

- **选择图像**（*必需）
- 提示词（可选）
- 图像标签（可选）
- 图像备注（可选）

#### 3.2.5 图像详情界面

- **左侧**: 提示词信息（标题、内容、翻译、备注、标签）
- **中间**: 图像显示、导航按钮
- **右侧**: 图像信息（文件名、标签、备注、安全评级开关（与关联提示词双向联动））

#### 3.2.6 设置界面

- **外观**: 主题切换
- **数据存储**: 数据目录更改
- **数据管理**: 导入、导出、清空数据

## 5. 非功能需求

### 5.1 性能要求

- 首次加载时间 < 3秒
- 搜索响应时间 < 500ms
- 图像缩略图生成 < 2秒

### 5.2 兼容性

- 支持 Windows 10/11
- 支持常见图像格式: JPG, PNG, GIF, WebP
- **JavaScript**: ES2023 完整支持 (ES Module, 可选链 `?.`, 空值合并 `??` 等)
- **Node.js**: >= 18.18.0
- **Electron**: 28.0.0 (Chromium 120+, V8 12.0+)

### 5.3 安全性

- 本地数据存储，不上传云端
- 支持安全评级控制内容显示

### 5.4 可维护性

- 代码结构清晰，模块化设计
- 数据库支持迁移
- 配置数据与业务数据分离

## 6. 开发规范

### 6.1 代码规范

- 使用 ES6+ 语法
- 异步操作使用 async/await
- 数据库字段使用 snake_case，前端数据使用 camelCase
- 控制台只输出英文
- 新添加的注释尽量简洁

### 6.2 文档规范

- 生成的 Markdown 格式文本要符合 markdownlint 要求
- 更改内容与 README.md 不一致时，提示更新 README.md

### 6.3 提交规范

- 功能提交: `feat: 描述`
- 修复提交: `fix: 描述`
- 文档提交: `docs: 描述`

### 6.4 开发流程

- 添加功能时，如有更好的想法，先与用户商量
- 修改代码前，确认是否违反需求文档
- 修改后，验证实现是否符合需求
- 需要启动应用时，提示用户操作
- 需要新的 node_modules 时，提示用户安装

### 6.5 文件组织

```text
project/
├── main.js           # 主进程入口 (ES Module)
├── preload.cjs       # 预加载脚本 (CommonJS)
├── database.js       # 数据库操作 (ES Module)
```
